    # JavaScript常见面试题

    # 1、let、const、var的区别和联系

    # var:

    • var是ES5声明变量的语法
    • 作用域为方法作用域，可以声明多个重复的变量
    • 声明变量但未赋值时，变量值为undefined
    • var声明的变量会进行声明提升(可以访问到变量，但是为undefined)

    # let：

    • let为ES6声明变量的方式
    • let不允许声明两个重复的变量
    • 作用域为块级作用域

    # const：

    • const为常量声明的方式，声明后必须进行初始化
    • const不允许对变量进行修改，或者说是对指向变量的引用的改变
    • 拥有和let一样的作用域，也不允许声明两个重复的变量

    # 2、undefined与null

    • undefined表示没有任何值,表示变量未被赋值时的属性；null表示没有任何对象，typeof null为object

      • 判断某些属性是否赋值时用let abc; abc === undefined
      • undefined在布尔类型环境中为false，在数值类型中为NaN
      • null在数值类型中为0，在布尔类型中为false

    • 当某些东西没有值时，一般表示undefined

      • 函数没有返回值时，一般默认返回undefined
      • 变量未初始化时为undefined
      • 访问对象上不存在的属性时为undefined
      • Array.prototype.find() (opens new window) 和 Map.prototype.get() (opens new window) 的方法，当没有发现元素时，返回 undefined

    # 3、forEach和map

    • forEach可以进行数组遍历，没有返回值，也不会影响原始对象
      • forEach不可被break
      • 执行时会自动跳过undefined值
      • [1,2,3].forEach((element,index,array) => console.log(ele,index,array))
    • map遍历数组，调用回调函数，返回新数组
      • [1,2,3].map((element,index,array) => element * 2)

    # 4、filter

    • 该方法为数组的过来方法，保留回调为true的值
    • 返回数组的一部分浅拷贝
      • [1,2,3,6,7,8].filter((element,index,array) => element > 4)

    # 8、Promise的执行顺序问题

    js处理任务分为两种，一是同步任务，二是异步任务。其中异步任务又分为异步微任务和异步宏任务。执行顺序同步任务>异步微任务>异步宏任务

    定时器等（setTimeout/setInterval）属于异步宏任务，由宿主机发起执行，Promise.then/catch/finally()属于异步微任务，由js发起执行

    JavaScript的事件循环机制：

    1、所有的同步任务都在主线程上执行，形成执行栈。

    2、主线程外还存在任务队列，包括宏任务队列和微任务队列，只要异步操作执行结束就会往对应的任务队列添加任务

    3、只要执行栈清空(主线程上的同步任务执行完成)，js引擎就会读取并执行微任务队列中的所有任务，清空微任务队列后，再去读取并执行宏任务队列中的任务

    主线程不断的重复执行上述第三个操作，这就是事件循环(Event Loop)

    再看Promise的执行顺序：

    setTimeout(()=>{
   console.log(1) 
})
new Promise((resolve)=>{
    console.log(2)
    resolve()
    console.log(5)
}).then(()=>{
   console.log(3) 
})
console.log(4) 
// 2 5 4 3 1

    # 9、闭包

    闭包类似于类可以有自己的私有作用空间，并且可以延长变量的生命周期。闭包是方法里返回方法，在没有出现es6的class之前的隔离解决方案，闭包会常驻内存。

    两个特性：

    • 变量隔离，形成私有空间
    • 延长变量的生命周期，一般函数执行完即销毁函数作用域内的变量，但是闭包会保留

    vue中的data被设计为一个函数的目的：

    • 其实data()就是一个闭包，形成每个vue文件的数据隔离

    数据隔离例子：

    function goods(){
    let num = 1
    function add(){
        num++
    }
    return {
        ad: function(){
            add()
        },
        num: function(){
            return num
        }
    }
}
const goods1 = goods()
const goods2 = goods()
goods2.ad()

goods1.ad()
goods1.ad()

console.log(goods1.num())
console.log(goods2.num())
//结果为:  3 2

    # 11、 原型与原型链

    # 原型

    每个实例对象都有一个私有属性(__proto__)称之为隐式原型，指向构造函数中的显式原型prototype，该原型对象也有一个自己的原型对象__proto__，指向上一层的原型对象，直到为null,一般为顶层(Object)

    # 原型链

    当对象需要寻找一个方法或属性时，就会先在自己身上找，如果没有就去原型上找，层层往上形成一个查找链。

    因为对象的原型对象层层链接，形成一个原型链。

    # 12、作用域

    代码执行的上下文，值和表达式在其中能否被访问或可见。子作用域可以访问父级作用域，反之则不行。

    作用域分为一下三种：

    • 全局作用域：代码运行的默认作用域
    • 模块作用域：模块模式中运行代码的作用域
    • 函数作用域：由函数创建的作用域

    还有一种由let、const声明的变量的作用域：

    • 块级作用域：用一对花括号（一个代码块）创建出来的作用域

    # 14、跨域的几种解决方案

    # 什么是跨域：

    同源策略是一种安全协议，是浏览器为了防止他人恶意攻击网站。协议：域名：端口相同即为同源。

    所以当涉及到非同源策略的数据交互时就需要跨域。

    # 跨域的几种解决方案

    • jsonp

      • 动态的插入script标签，利用url进行数据加载
      • 只能发送get请求
      • 好像百度有在用

    • 前端反向代理

      • 开发时利用webpack或者vite配置代理

      • proxy: {
    '/api': { // 配置匹配规则/api
        target: 'http://ad.server.com', 	//配置目标地址
        changeOrigin: true, 	// 是否允许跨域
        pathRewrite: {
            '^/api': '/',    //配置路径重写
        },
    }
}

      • 部署时可利用nginx进行代理

    • CORS（跨域资源共享）后端配置

      • 后端(服务器端)设置 Access-Control-Allow-OriginHTTP 响应头即可
