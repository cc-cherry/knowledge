# CSS相关

# 响应式布局和自适应布局

参考：https://juejin.cn/post/6844903814332432397

响应式布局是通过媒体查询来实现不同屏幕尺寸下的布局变化，可以根据屏幕大小、分辨率、方向等因素自动调整布局。（bootstrap的sm/md/lg/xl/xxl等）

自适应布局则是根据浏览器窗口尺寸来调整布局，一般是在设计中指定几个固定的布局尺寸、然后根据浏览器窗口尺寸选择相应的布局来显示

响应式布局和自适应布局都是为了提供不同屏幕尺寸下的最佳用户体验，在不同设备上自适应地展示网站内容。它们都是基于CSS技术实现的，并且都需要设计师考虑视觉效果、用户体验、交互性等方面的问题。

# 响应式布局（媒体查询）

通过监测视口的分辨率，对不同设备进行处理。移动端优先首先使用的是min-width，PC端优先使用的max-width。

在媒体查询下的实现方式：

1、静态像素：**缺点：**代码多

2、百分比布局: **缺点：**计算困难，如果我们要定义一个元素的宽度和高度，按照设计稿，必须换算成百分比单位

3、rem布局(css3，目前多屏适配的最佳方式，默认根font-size:16px) **缺点：**和js耦合

​ rem单位是相对根元素html的font-size来决定大小，根据视图容器的大小，动态的改变根font-size的大小(em是相对父元素的)

4、视口单位(css3) **缺点：**兼容性问题

​ vw表示视口的宽度，vh表示视口的高度

5、图片响应式

​ 1、使用max-width:100%;height:auto;display:inline-block

​ 2、使用srcset

​ img srcset="photo_w350.jpg 1x, photo_w640.jpg 2x" src="photo_w350.jpg" alt="">

​ 屏幕dpi=1，会加载1倍图，dpi=2，加载2倍图。兼容性不好mac和chrome会加载两张图片

​ 3、background-image

​ 4、使用picture标签

​ picturefill.min.js (opens new window) ：解决IE等浏览器不支持 的问题

<picture>
    <source srcset="banner_w1000.jpg" media="(min-width: 801px)">
    <source srcset="banner_w800.jpg" media="(max-width: 800px)">
    <img src="banner_w800.jpg" alt="">
</picture>

<!-- picturefill.min.js 解决IE等浏览器不支持 <picture> 的问题 -->
<script type="text/javascript" src="js/vendor/picturefill.min.js"></script>

# 1）成型的布局方案

现在的css，UI框架等都已经考虑到了适配不同屏幕分辨率的问题，实际项目中我们可以直接使用这些新特性和框架来实现响应式布局。可以有以下选择方案：

  • 利用上面的方法自己来实现，比如CSS3 Media Query,rem，vw等
  • Flex弹性布局，兼容性较差
  • Grid网格布局，兼容性较差
  • Columns栅格系统，往往需要依赖某个UI库，如Bootstrap

# 2）要点

在实际项目中，我们可能需要综合上面的方案，比如用rem来做字体的适配，用srcset来做图片的响应式，宽度可以用rem，flex，栅格系统等来实现响应式，然后可能还需要利用媒体查询来作为响应式布局的基础，因此综合上面的实现方案，项目中实现响应式布局需要注意下面几点：

  • 设置viewport
  • 媒体查询
  • 字体的适配（字体单位）
  • 百分比布局
  • 图片的适配（图片的响应式）
  • 结合flex，grid，BFC，栅格系统等已经成型的方案

# 变量的定义和使用

定义： --main-text-color 使用--开头

使用：var(--main-text-color)

​ 可以有默认值var(--main-text-color, red)

​ 当变量的值无效时，会先继承父元素的值，如没有则使用默认值
