    # 初识Webpack

    Webpack的基本原理是将所有的资源视为模块，并通过不同的loader和插件对这些模块进行处理和转换，最终将它们打包成一个或多个浏览器可识别的文件

    Webpack通过一个入口文件来启动打包过程，根据入口文件的依赖关系，递归地构建整个应用程序的依赖图谱，并将每个依赖项都打包成一个单独的模块。

    Webpack的核心是其插件(Plugin)系统和loader系统。插件系统提供了各种不同的功能，例如代码压缩、热加载、代码分离等，而loader系统则用于处理各种类型

    的文件和资源，例如CSS、图片、字体等。Webpack的输出文件可以是多种不同的格式，例如JavaScript、CSS、HTML等。Webpack还支持开发模式和生产模

    式，开发模式可以帮助开发者快速构建应用程序，而生产模式则可以优化应用程序的性能和大小。

    # Loader

    Loader用于处理各种类型的文件和资源，例如CSS、图片、字体、vue文件等。在Webpack中，每个模块都可以通过一个或多个loader进行处理和转

    换，最终将处理后的模块打包到输出文件中。

    基本流程：

    配置loader => 解析模块 => 转换代码 => 打包模块

    1、配置loader

    在Webpack的配置文件中，可以通过在module.rules属性中配置loader。每个loader都是一个对象，包含test、use等属性。test用于匹配需要处理的文件类型大

    use则是一个数组，包含了需要使用的loader。

    2、解析模块

    在Webpack的打包过程中，每个模块都会经过loader的处理。当Webpack遇到需要处理的模块时，它会使用对应的loader将模块转换成JavaScript的模块代码。

    3、转换代码

    loader会将模块的内容解析成JavaScript代码，并将其返回给Webpack。这些代码可以是ES6、TypeScript、CoffeeScript等不同的语言，也可以是CSS、

    SCSS、LESS等不同的样式表。

    4、打包模块

    Webpack将所有经过loader处理后的模块进行打包，生成最终的输出文件。在打包过程中，Webpack会将所有的模块按照其依赖关系进行排序，并生成一个或多

    个浏览器可识别的文件。

    # 常见的loader:

    1、babel-loader：用于将ES6及以上版本的JavaScript代码转换成ES5的代码，兼容更久的浏览器。

    2、css-loader：用于处理CSS文件，并解析其中的import语句和url()函数

    3、style-loader：将CSS代码以style标签的形式注入到HTML文档中

    4、sass-loader：用于处理sass/scss文件，并将其转换成css代码

    5、less-loader：用于处理less文件，并将其转换成css代码

    6、postcss-loader：用于使用PostCSS插件对CSS代码进行处理和转换

    7、file-loader：用于处理文件，并将文件输出到文件夹中

    8、url-loader：类似file-loader，但可以将小文件转换成DataURL(base64)，减少HTTP请求

    9、json-loader：用于处理JSON格式的数据，将其转换成Javascript的对象或数组

    10、xml-loader：用于处理xml格式的数据，将其转换成Javascript对象或数组

    11、vue-loader：用于处理vue文件，将其转换成Javascript模块，以便webpack能够将他们打包成一个或多个浏览器能够识别的文件。把不同模块的代码交给不同的loader进行解析。

    12、mini-css-extract-plugin：用于将CSS代码提取为单独的文件，以便在浏览器中进行缓存和优化。

    常见css-loader的顺序：[style-loader,postcss-loader,css-loader,sass-loader]

    # Loader原理

    执行顺序： pre(前置loader) > normal(普通loader) > inline(内联loader) > post(后置loader)

    module.exports = function (content, map, meta) {

    # Loader分类:

    同步Loader、异步Loader、RawLoader、PitchingLoader

    1、同步loader

    module.exports = function (content, map, meta) {
  return content;
};
// 回调的方式可以传递多个参数
module.exports = function (content, map, meta) {
  // 传递map，让source-map不中断
  // 传递meta，让下一个loader接收到其他参数
  this.callback(null, content, map, meta);
  return; // 当调用 callback() 函数时，总是返回 undefined
};

    2、异步loader

    module.exports = function (content, map, meta) {
  const callback = this.async();
  // 进行异步操作
  setTimeout(() => {
    callback(null, result, map, meta);
  }, 1000);
};

    3、RawLoader（文件资源相关的loader）

    默认情况下，资源文件会被转化为 UTF-8 字符串，然后传给 loader。通过设置 raw 为 true，loader 可以接收原始的 Buffer。

    module.exports = function (content) {
  // content是一个Buffer数据
  return content;
};
module.exports.raw = true; // 开启 Raw Loader

    4、PitchingLoader

    module.exports = function (content) {
  return content;
};
//会先执行
module.exports.pitch = function (remainingRequest, precedingRequest, data) {
  console.log("do somethings");
};

    webpack 会先从左到右执行 loader 链中的每个 loader 上的 pitch 方法（如果有），然后再从右到左执行 loader 链中的每个 loader 上的普通 loader 方法。在这个过程中如果任何 pitch 有返回值，则 loader 链被阻断。webpack 会跳过后面所有的的 pitch 和 loader，直接进入上一个 loader 。

    # 手写Loader

    clean-log-laoder（清除日志输出）、banner-loader（给js代码添加文本）

    babel-loader（编译js代码，将ES6+语法装换成ES5-语法）

    // npm i @babel/core @babel/preset-env -D
const schema = require("./schema.json");
const babel = require("@babel/core");

module.exports = function (content) {
  const options = this.getOptions(schema);
  // 使用异步loader
  const callback = this.async();
  // 使用babel对js代码进行编译
  babel.transform(content, options, function (err, result) {
    callback(err, result.code);
  });
};

    file-loader（将文件原封不动输出出去）

    // npm i loader-utils -D
const loaderUtils = require("loader-utils");
function fileLoader(content) {
  // 根据文件内容生产一个新的文件名称
  const filename = loaderUtils.interpolateName(this, "[hash].[ext]", {
    content,
  });
  // 输出文件
  this.emitFile(filename, content);
  // 暴露出去，给js引用。
  // 记得加上''
  return `export default '${filename}'`;
}
// loader 解决的是二进制的内容
// 图片是 Buffer 数据
fileLoader.raw = true;

module.exports = fileLoader;

    # Plugin

    Webpack的plugin是一种可以扩展Webpack功能的机制，它可以在Webpack构建过程中的不同阶段执行自定义的逻辑，例如优化代码、生成额外的资源文件等。

    Plugin的原理是通过Webpack的事件钩子（hook）机制实现的，Webpack在构建过程中会触发一系列的事件钩子，Plugin可以注册自己的回调函数，在事件被触

    发时执行自定义的逻辑。每个事件钩子都有对应的事件参数，Plugin可以通过这些参数获取Webpack的状态信息，从而进行更加精细的控制和优化。

    Tapable 还统一暴露了三个方法给插件，用于注入不同类型的自定义构建行为：

    • tap：可以注册同步钩子和异步钩子。
    • tapAsync：回调方式注册异步钩子。
    • tapPromise：Promise 方式注册异步钩子。

    # Compiler

    compiler 对象中保存着完整的 Webpack 环境配置，每次启动 webpack 构建时它都是一个独一无二，仅仅会创建一次的对象。

    这个对象会在首次启动 Webpack 时创建，我们可以通过 compiler 对象上访问到 Webapck 的主环境配置，比如 loader 、 plugin 等等配置信息。

    它有以下主要属性：

    • compiler.options 可以访问本次启动 webpack 时候所有的配置文件，包括但不限于 loaders 、 entry 、 output 、 plugin 等等完整配置信息。
    • compiler.inputFileSystem 和 compiler.outputFileSystem 可以进行文件操作，相当于 Nodejs 中 fs。
    • compiler.hooks 可以注册 tapable 的不同种类 Hook，从而可以在 compiler 生命周期中植入不同的逻辑。

    # Compilation

    compilation 对象代表一次资源的构建，compilation 实例能够访问所有的模块和它们的依赖。

    一个 compilation 对象会对构建依赖图中所有模块，进行编译。 在编译阶段，模块会被加载(load)、封存(seal)、优化(optimize)、 分块(chunk)、哈希(hash)和重新创建(restore)。

    它有以下主要属性：

    • compilation.modules 可以访问所有模块，打包的每一个文件都是一个模块。
    • compilation.chunks chunk 即是多个 modules 组成而来的一个代码块。入口文件引入的资源组成一个 chunk，通过代码分割的模块又是另外的 chunk。
    • compilation.assets 可以访问本次打包生成所有文件的结果。
    • compilation.hooks 可以注册 tapable 的不同种类 Hook，用于在 compilation 编译模块阶段进行逻辑添加以及修改。

    1）CleanWebpackPlugin

    打包前清空上次打包内容。

    思路：

    • 如何在打包输出前执行？需要使用 compiler.hooks.emit 钩子, 它是打包输出前触发。
    • 如何清空上次打包内容？
      • 获取打包输出目录：通过 compiler 对象。
      • 通过文件操作清空内容：通过 compiler.outputFileSystem 操作文件。

    2）AnalyzeWebpackPlugin

    分析webpack打包资源的大小，并输出分析文件

    思路： compiler.hooks.emit, 它是在打包输出前触发，我们需要分析资源大小同时添加上分析后的 md 文件。

    # 优化手段

    **1、减少代码体积：**代码/图片压缩( UglifyJSPlugin插件)、去除未使用代码( tree shaking )、减少打包文件体积、开启gzip压缩

    **2、按需加载：**使用代码webpack分割(code splitting - SplitChunksPlugin插件 )。减少页面加载时间

    3、使用缓存： 使用HardSourceWebpackPlugin插件或者使用webpack的缓存机制，可以减少webpack的构建时间。使用cache-loader或者hard-source-webpack-plugin插件进行缓存

    **4、去除不必要的插件：**提高构建效率和打包速度

    **5、使用多线程构建：**使用webpack的ParallelUglifyPlugin等插件，可以将构建过程分配到多个子进程中，从而加快构建速度。

    6、静态资源CDN加速： 将静态资源上传至CDN，从而减少加载时间，提高页面的响应速度。

    7、preload和prefetch(立即加载或空闲加载)、polyfill打补丁(core-js)、PWA(离线运行，基于Service Workers)
